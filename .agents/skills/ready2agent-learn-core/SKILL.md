# Ready2Agent Learn Core

## Purpose

Definire il contratto operativo base per lavorare su `Ready2Agent Learn` mantenendo chiarezza didattica, scope corretto e documentazione coerente.

## When to use

- ogni task su `README.md`, `documentation/`, `prompts/`, `workflow/`, `.agents/skills/`
- ogni task che cambia istruzioni per coding agent o flussi di utilizzo

## Repository nature

- laboratorio open-source didattico
- orientato a metodo, apprendimento e verifica
- non e il framework operativo enterprise completo

## Owns

- confini didattici del repository
- coerenza tra documentazione, prompt e workflow
- linguaggio tecnico sobrio e verificabile
- disciplina quality gate in base agli script reali

## Does not own

- runtime enterprise completo
- moduli business reali o codice cliente
- marketing/sales copy
- migrazioni stack non richieste

## Editing rules

- implementa un task alla volta
- non introdurre feature fuori scope
- non reintrodurre componenti enterprise vietati
- dichiara decisioni e trade-off nel report finale

## Documentation rules

- ogni modifica deve spiegare cosa fare e perche
- evita hype, slogan e promesse commerciali
- evita duplicazioni inutili tra README e documentazione interna
- mantieni link e navigazione coerenti

## Skills rules

- le skills di `learn` sono modificabili dagli utenti
- le skills possono essere unite o ridotte
- non serve replicare tutte le skills operative
- preferisci poche skills, corte e ad alta utilita

## OpenCode/local setup rules

- usa esempi multipiattaforma (Linux, macOS, Windows)
- non usare path personali
- documenta come caricare le skills da `.agents/skills/`
- esplicita differenza tra `Ready2Agent Learn` e repository operativo

## Final reporting checklist

- scope confermato e confini rispettati
- file modificati/eliminati/creati elencati
- documentazione allineata (README + quickstart + indici rilevanti)
- esito quality gate riportato (o check manuali motivati)
- rischi residui e prossimi step dichiarati
