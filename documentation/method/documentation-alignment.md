# Documentation Alignment

## Cosa significa

Tenere allineati README, documentazione operativa e implementazione reale.

## Perche serve

Se documentazione e codice divergono, il team prende decisioni su informazioni sbagliate.

## Come usarlo con un agente

A fine task esegui `prompts/06-documentation-alignment.md` per trovare gap e applicare fix.

## Esempio pratico

Prendi una modifica in `src/app/page.tsx` e verifica che README, mappe e guide restino coerenti.

## Errori comuni

- aggiornare solo il codice
- link rotti tra sezioni
- struttura repository non riflessa nei documenti

## Checklist

- README aggiornato
- hub documentazione aggiornati
- riferimenti a file reali
- quality gate documentale eseguito
