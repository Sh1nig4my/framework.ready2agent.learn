# Prompt Discipline

## Cosa significa

Scrivere prompt come istruzioni operative, non come desideri generici.

## Perche serve

Un prompt debole produce output difficile da validare.

## Come usarlo con un agente

Usa sempre: ruolo, task, input, output, vincoli, quality check.

## Esempio pratico

Parti da `playground/04-business-context-to-agent-prompt.md` e verifica con `documentation/examples/bad-vs-good-prompts.md`.

## Errori comuni

- task multipli nello stesso prompt
- assenza di output atteso
- vincoli mancanti

## Checklist

- task unico
- input concreti
- output verificabile
- vincoli espliciti
- quality check definiti
