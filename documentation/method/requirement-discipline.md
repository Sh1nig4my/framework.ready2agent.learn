# Requirement Discipline

## Cosa significa

Definire cosa vogliamo ottenere prima di discutere come implementarlo.

## Perche serve

Senza requirement chiaro, il coding agent inventa parti mancanti.

## Come usarlo con un agente

Chiedi output con sezioni obbligatorie: obiettivo, scope, out-of-scope, vincoli, criteri, rischi.

## Esempio pratico

Usa `playground/02-chaotic-idea-to-requirement.md` e confronta il risultato con `documentation/examples/bad-vs-good-requirements.md`.

## Errori comuni

- confondere requirement con soluzione
- non dichiarare out-of-scope
- criteri non misurabili

## Checklist

- obiettivo in 1-2 frasi
- scope esplicito
- out-of-scope esplicito
- criteri verificabili
- rischi principali
