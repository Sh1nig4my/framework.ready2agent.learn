# Repository Split

## framework.ready2agent

Repository privato, enterprise, operativo.

Contiene implementazioni reali, codice completo e asset applicativi.

## framework.ready2agent.learn

Repository pubblico, open-source, formativo.

Contiene metodo, workflow, prompt, landing e caso studio guidato.

## Perche la separazione

- proteggere asset operativi
- rendere pubblico il metodo
- facilitare formazione
- creare una base divulgativa
- evitare confusione tra prodotto e laboratorio
