# Non Technical Start Here

## Puoi usare questo repository anche senza essere sviluppatore

Si. Ready2Agent Learn non e solo codice: e un laboratorio per imparare un metodo. Puoi partire dalla logica del processo e usare un coding agent come assistente, anche se non scrivi codice ogni giorno.

## Glossario minimo

- **Repository**: la cartella completa del progetto.
- **README**: la guida principale di una sezione.
- **Markdown**: formato testuale usato per la documentazione (`.md`).
- **Prompt**: istruzione strutturata per l'agente.
- **Agente di coding**: AI che legge file e propone/modifica codice.
- **Requisito**: obiettivo e confini del lavoro.
- **Workflow**: percorso ordinato dal requisito alla validazione.
- **Task**: passo operativo piccolo e verificabile.
- **Quality gate**: controlli minimi per validare il risultato.

## Tre modi di usare il repository

1. **Solo studio**: leggi metodo, workflow e prompt per imparare la disciplina.
2. **Studio + agente**: fai analizzare i documenti all'agente e chiedi spiegazioni.
3. **Implementazione tecnica**: usi prompt e workflow per costruire step-by-step.

## Percorso da 30 minuti

1. leggi `README.md`
2. apri la landing locale
3. leggi questa guida
4. apri `prompts/README.md`
5. copia il prompt standard di avvio in Codex

## Percorso da 2 ore

1. leggi `documentation/method/README.md`
2. leggi `workflow/README.md`
3. leggi il master requirement Auth/IAM
4. chiedi a Codex di spiegarti il caso studio
5. chiedi a Codex di generare un piano operativo

## Percorso da piu giorni

1. segui i prompt in ordine
2. genera task incrementali
3. implementa o fai implementare
4. valida con quality gate
5. riallinea documentazione e codice

## Come capire se l'agente sta lavorando bene

- cita i file che ha letto
- esplicita assunzioni e limiti
- propone step piccoli
- non inventa feature extra
- produce output verificabili
- aggiorna anche la documentazione
