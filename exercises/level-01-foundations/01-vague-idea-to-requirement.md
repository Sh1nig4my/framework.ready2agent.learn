# Vague Idea to Requirement

## Livello

1 - Foundations

## Obiettivo

Trasformare un'idea vaga in requirement verificabile.

## Perche e importante

Senza requirement chiaro, il coding agent amplifica ambiguita.

## Scenario

"Vorrei una dashboard per gestire clienti e attività".

## Input disponibili

- idea iniziale
- target utente
- vincoli tempo/stack

## Task richiesto

Scrivi requirement con obiettivo, scope, out-of-scope, criteri successo, rischi.

## Vincoli

- niente soluzione tecnica dettagliata
- frasi concrete

## Prompt consigliato

`prompts/01-requirement-analysis.md`

## Output atteso

Documento requirement breve, leggibile, verificabile.

## Criteri di valutazione

- scope e out-of-scope distinti
- criteri misurabili
- rischi esplicitati

## Errori comuni

- confondere obiettivo con implementazione
- criteri vaghi

## Aggiorna il tracker

Aggiorna `exercises/tracker.md` con stato, output e note.

## Variante avanzata

Aggiungi roadmap in 3 fasi con dipendenze.

## Checklist di completamento

- [ ] requirement con obiettivo verificabile
- [ ] in-scope e out-of-scope espliciti
- [ ] rischi principali dichiarati
- [ ] scorecard compilata
