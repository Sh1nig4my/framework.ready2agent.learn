# Task Breakdown
## Livello
2 - Agent Workflow
## Obiettivo
Spezzare una richiesta grande in task eseguibili e verificabili.
## Perche e importante
Task piccoli riducono regressioni e review costose.
## Scenario
Feature complessa senza piano task.
## Input disponibili
Requirement e architettura target.
## Task richiesto
Crea backlog task con dipendenze e DoD.
## Vincoli
Massimo 1 obiettivo per task.
## Prompt consigliato
`prompts/03-task-breakdown.md`
## Output atteso
Backlog ordinato pronto per implementazione.
## Criteri di valutazione
Dipendenze esplicite e DoD verificabile.
## Errori comuni
Task monolitici.
## Aggiorna il tracker
Inserisci task principali e stato.
## Variante avanzata
Aggiungi stima rischio/complessita.

`04-build-workflow-tracker.md`
