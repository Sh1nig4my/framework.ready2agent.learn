# Define Quality Gate
## Livello
2 - Agent Workflow
## Obiettivo
Definire gate minimi per chiudere un task.
## Perche e importante
Senza gate chiudi task non affidabili.
## Scenario
Consegna veloce ma non validata.
## Input disponibili
Task completato e diff previsto.
## Task richiesto
Definisci gate tecnici, strutturali e documentali.
## Vincoli
Gate eseguibili, no checklist finta.
## Prompt consigliato
`prompts/05-quality-gate.md`
## Output atteso
Schema quality gate con esito pass/fail.
## Criteri di valutazione
Gate ripetibili e misurabili.
## Errori comuni
Saltare gate documentali.
## Aggiorna il tracker
Registra gate passati/falliti.
## Variante avanzata
Aggiungi policy per gestione fallimenti.

## Checklist di completamento

- [ ] gate tecnici, strutturali e documentali definiti
- [ ] esito pass/fail tracciabile
- [ ] piano fix in caso di failure
- [ ] scorecard compilata
