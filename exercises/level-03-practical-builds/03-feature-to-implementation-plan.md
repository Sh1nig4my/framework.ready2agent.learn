3 - Practical Builds

Preparare piano tecnico incrementale da una singola feature.

Riduce rischi di task monolitici.
## Scenario
Nuova feature con dipendenze trasversali.
## Input disponibili
Feature brief, file target, vincoli.
## Task richiesto
Crea piano task e DoD.
## Vincoli
Task verticali e piccoli.
## Prompt consigliato
`playground/03-feature-to-codex-plan.md`
## Output atteso
Piano pronto per implementazione.
## Criteri di valutazione
Ordine logico, dipendenze, criteri done.
## Errori comuni
Task senza validazione.
## Aggiorna il tracker
Aggiorna decisioni.
## Variante avanzata
Piano conservativo vs rapido.

| Dimensione | 0 | 1 | 2 | 3 |
