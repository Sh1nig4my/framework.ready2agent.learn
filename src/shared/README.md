# Shared Skeleton

`shared/` contiene contratti e tipi comuni tra frontend e server. Nel metodo Ready2Agent e il punto che mantiene allineato il linguaggio tecnico del progetto.

## Perche e importante

Quando frontend e backend usano definizioni diverse, nascono mismatch difficili da tracciare. Centralizzare i contratti evita ambiguita e aiuta l'agente a produrre output coerenti.

## Cosa trovi qui

- `contracts/`: area dedicata a payload e tipi condivisi del caso studio
